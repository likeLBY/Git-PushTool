# Git 推送工具 - Aby
一款简洁美观的 Git 图形化管理工具，专为不熟悉命令行的用户设计。支持 Windows 系统，让 Git 操作变得简单直观。

## ✨ 功能特点

- 🎨 现代化 UI 设计，支持深色/浅色主题切换
- 🌏 完整中文界面，对中文路径和文件名友好
- 📁 支持拖拽文件夹快速打开仓库
- 🔄 实时监控文件变化，有时候有bug就点一下刷新
- 💾 最近打开的仓库记录
- 🎯 一键提交推送，简化工作流程
- 📝 标题栏显示一言（名言），每5秒自动刷新

| 按钮  | 提示                | 功能说明              |
| --- | ----------------- | ----------------- |
| ↩️  | 切换到此分支            | 切换当前工作分支          |
| 🔀  | 将「XXX」分支合并到「当前分支」 | 把选中分支的代码合并到当前分支   |
| 📥  | 将「XXX」分支替换到「当前分支」 | 用选中分支完全覆盖当前分支（危险） |
| 🗑️ | 删除分支              | 删除该分支（会弹出确认框）     |

### 合并 vs 替换 的区别
  
**合并（Merge）：**
- 将两个分支的修改合并在一起
- 保留两边的提交历史
- 如果有冲突需要手动解决
- 适合：团队协作，保留完整历史

**替换（Reset）：**
- 用目标分支完全覆盖当前分支
- 当前分支的独有修改会丢失
- 不会产生冲突
- 适合：放弃当前分支的修改，重新开始

## 🗑️ 清理大文件面板
点击工具栏的 🗑️ 图标打开。

  ![](images/Pasted%20image%2020260117114459.png)
### 功能
查找 Git 历史中超过 50MB 的大文件。这些文件可能导致：
- 推送到 Gitee/GitHub 失败（平台有文件大小限制）
- 仓库体积过大
### 操作

| 按钮     | 功能               |
| ------ | ---------------- |
| 「刷新」   | 重新扫描大文件          |
| 「清理」   | 从 Git 历史中彻底删除该文件 |
| 「强制推送」 | 清理后需要强制推送到远程     |

⚠️ **警告**：清理大文件会重写 Git 历史，需要强制推送，可能影响其他协作者。

## 🚀 快速上手
### 第一次使用
1. **安装 Git**（如果没有）
   - 软件左下角会提示「没有安装 Git？」
   - 点击图标运行内置安装程序
1. **配置用户信息**
   - 点击用户名旁边的 ⚙️
   - 输入你的用户名和邮箱
   - 勾选「全局配置」
   - 点击「保存」
1. **打开仓库**
   - 点击「打开仓库」选择文件夹
   - 或直接拖拽文件夹到窗口
### 日常使用流程
1. **修改文件** - 在你喜欢的编辑器中修改代码
2. **查看更改** - 软件会自动显示更改的文件
3. **暂存文件** - 点击「全部暂存」或单独暂存
4. **输入描述** - 在底部输入框写提交信息
5. **提交推送** - 点击「一键提交推送」完成
### 快捷操作

| 操作     | 方法            |
| ------ | ------------- |
| 快速提交推送 | 「一键提交推送」按钮    |
| 批量暂存   | 勾选多个文件后点击「暂存」 |
| 撤销所有更改 | 「全部撤销」按钮      |
| 查看文件改动 | 点击文件名         |

## ❓ 常见问题
### Q: 推送失败怎么办？
**A:** 常见原因：
1. 网络问题 - 检查网络连接
2. 远程有更新 - 先点击「拉取」再推送
3. 文件过大 - 使用「清理大文件」功能
4. 认证失败 - 检查 Git 凭据

### Q: 中文文件夹/文件名显示乱码？
本软件已优化中文支持，如仍有问题请反馈。
### Q: 如何解决合并冲突？
1. 合并后如有冲突，文件会显示在更改列表
2. 手动编辑冲突文件，解决冲突标记
3. 暂存解决后的文件
4. 提交完成合并
### Q: 误删文件怎么恢复？
- 如果还没提交：使用「撤销更改」
- 如果已提交：在「提交历史」中撤销该提交
